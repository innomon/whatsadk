# Conductor Tracks

This file acts as a registry for the development tracks of the WhatsADK project.

- [x] [adk-silent-ignore](tracks/adk-silent-ignore/spec.md) - Support for silencing ADK replies.
- [x] [adk-simulator](tracks/adk-simulator/) - TUI simulator for ADK API responses.
- [x] [adk2whatsapp-media](tracks/adk2whatsapp-media/) - Forwarding outbound media from ADK to WhatsApp.
- [x] [cron-heartbeat](tracks/cron-heartbeat/) - Gateway-driven cron heartbeats.
- [x] [logging](tracks/logging/spec.md) - Structured logging and log rotation.
- [x] [mcp-autonomous-agent](tracks/mcp-autonomous-agent/) - Running external agents over MCP in autonomous mode.
- [x] [mcp-filesys](tracks/mcp-filesys/) - Accessing DB filesys logs via MCP.
- [x] [mcp-server](tracks/mcp-server/) - Multiplexed command queue MCP server.
- [x] [media-bridge](tracks/media-bridge/) - Resizing and converting inbound WhatsApp media formats.
- [x] [reverse-otp](tracks/reverse-otp/plan.md) - Two-factor authentication via WhatsApp and signed callbacks.
- [x] [router](tracks/router/) - Router agent for multi-app routing.
- [x] [router-in-process](tracks/router-in-process/plan.md) - In-process agent execution for routing.
- [x] [waba-gateway](tracks/waba-gateway/plan.md) - Official WABA webhook and client gateway.
- [x] [whatsapp-auth](tracks/whatsapp-auth/) - SPA WhatsApp-based login using EdDSA JWTs.
- [x] [whatsapp-blocklist](tracks/whatsapp-blocklist/) - Unified shadow ban and WhatsApp blocklist management.
